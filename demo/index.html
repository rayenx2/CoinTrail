📄 Monthly PDF Statement

Generates a professional PDF statement via GET /api/reports/monthly?startDate=&endDate= — transactions, category summaries, and net balance for the selected date range. Rendered server-side with iText 7.

📊 CSV Export

Exports every transaction in a date range via GET /api/export/csv?startDate=&endDate= — UTF-8 BOM-prefixed, RFC 4180 compliant, with a running balance column. Opens cleanly in Excel, Google Sheets, or DATEV import tools.

Everything You Need to Track Finances

Built with production patterns: stateless JWT auth, Redis caching, PDF generation, and real-time analytics — self-hostable in one command.

🔐

JWT Auth with Refresh Rotation

Stateless authentication with 24-hour access tokens and 30-day refresh tokens. Axios interceptor silently renews sessions on 401 — no re-login required.

POST /api/auth/refresh-token
📊

Live Analytics Engine

Category breakdowns, savings rate, and 1–24 month income/expense trends computed from raw transactions in a single DB query — zero N+1 round-trips.

GET /api/analytics/summary
📄

PDF Monthly Statements

Enterprise-grade PDF generation via iText 7 in the same JVM process. No external service call required — generates in ~200 ms for 100 transactions.

GET /api/reports/monthly
⏰

Automated Budget Alerts

Spring @Scheduled cron runs at 09:00 daily. Checks all active budgets — dispatches email alerts at 90% and 100% spend thresholds.

cron: 0 0 9 * * ?
🔁

Recurring Transactions

Set rent, a subscription, or your salary once with a day-of-month. A separate daily job auto-creates the real transaction when it's due — tracked per month so it never double-fires.

cron: 0 5 0 * * ?
🗃️

Redis-Backed Report Cache

Monthly PDF statements are cached per user and date range with Spring Cache — regenerating the same statement twice hits Redis instead of re-querying and re-rendering. Cache is evicted automatically whenever a transaction or budget changes.

@Cacheable("monthlyReports")
🩺

Observability Built-in

Spring Actuator exposes /actuator/health and /actuator/metrics. Custom RequestLoggingFilter injects correlation IDs into every request log.

GET /actuator/health

Up and Running in 3 Commands

1

Clone and Configure

git clone https://github.com/rayenx2/CoinTrail.git && cd CoinTrail && cp .env.example .env — all defaults work out of the box, no API keys required.

2

Start All 4 Services

docker compose up -d --build — spins up backend (Spring Boot), frontend (Vite), PostgreSQL 15, and Redis 7. Backend becomes healthy in ~60 seconds.

3

Open the App

Frontend at http://localhost:5062 — register, log in, and start tracking. Swagger UI available at http://localhost:8060/swagger-ui.html.

API Reference

Method Endpoint Description Auth
POST /api/auth/register Register a new user —
POST /api/auth/login Login — returns access + refresh tokens —
POST /api/auth/refresh-token Rotate refresh token, get new access token —
GET /api/transactions Paginated transaction list JWT
POST /api/transactions Create new income or expense transaction JWT
PUT /api/transactions/{id} Update an existing transaction JWT
DELETE /api/transactions/{id} Delete a transaction JWT
GET /api/budgets List category budgets for current month JWT
POST /api/budgets Create or update a monthly category budget JWT
GET /api/categories List all expense and income categories JWT
GET /api/analytics/summary?months=6 Analytics: income, expenses, trends, category breakdown, savings rate JWT
GET /api/reports/monthly?startDate=&endDate= Download PDF monthly statement (iText 7) JWT
GET /api/export/csv?startDate=&endDate= Export transactions in a date range to CSV JWT
GET /api/users/me Get the current user's profile JWT
PATCH /api/users/me Update display name and default currency JWT
POST /api/users/me/change-password Change the current user's password JWT
POST /api/recurring-transactions Create a monthly recurring transaction rule JWT
GET /api/recurring-transactions List recurring transaction rules JWT
DELETE /api/recurring-transactions/{id} Stop a recurring transaction rule JWT
GET /actuator/health Service health: DB + Redis component status —
